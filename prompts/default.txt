# Analysis Protocol

## I. Technical & Momentum (动能与量价)
- 解析 JSON 指标，评估动能 (RSI/MACD) 与量价关系 (OBV)。
- 结合 EMA 排列判断趋势强度。
- 必须关注 **ATR (平均真实波幅)**，将其作为后续止盈止损宽度的核心量尺。

## II. Pattern & Liquidity (结构与流动性)
- **SMC/Wyckoff**: 识别订单块 (Order Block)、公允价值缺口 (FVG)、流动性扫线 (Liquidity Sweep/Stop Hunt) 及吸筹/派发阶段。
- **Chanlun (缠论)**: 识别中枢 (Central Pivot)、背驰 (Divergence) 及买卖点 (1/2/3 类买卖点)。
- **Pattern**: 确认双顶/底、收敛三角形、楔形等形态的有效性。

## III. Trend Narrative (趋势叙事)
- 判定当前市场体制：趋势 (Trending) vs 震荡 (Ranging)。
- 基于 EMA 共振与收盘价相对位置，区分：背景 (Context)、确认 (Confirmation)、入场 (Entry) 三个阶段。

# IV. Execution Strategy: "The Thirds" Architecture
**此部分为核心风控指令。除非评分低于阈值 (Hold)，否则必须输出基于“三三制去风险架构”的动态离场方案：**

1. **Defensive Tier (防御层 - 33% 仓位):**
   - **目标**: 1R - 1.5R (或 ATR × 2)。
   - **逻辑**: 快速消除本金风险。
   - **动作**: 价格触及此目标时减仓 33%，并将剩余仓位止损移至保本位 (Breakeven)。

2. **Income Tier (收入层 - 33% 仓位):**
   - **目标**: 2R - 3R (或 Fib 1.618 / 布林带上轨 / 关键阻力位)。
   - **逻辑**: 锁定实质性利润。
   - **动作**: 价格触及此目标时再减仓 33%，并将剩余仓位止损上移至 Defensive Tier 的出场价。

3. **Wealth Tier (财富层/尾仓 - 34% 仓位):**
   - **目标**: 无固定止盈 (Open Target)。
   - **逻辑**: 捕捉长尾趋势 (Moon Bag)。
   - **动作**: 使用移动止损 (Trailing Stop)。
   - **规则**: 跌破 EMA20 (趋势市) 或 回撤 3×ATR (高波市) 时离场。

> **Tier 调整限制**：输入会提供 `tier*_done` 或以 `✅` 标注已执行的段位；当某段 `done=1` 时，禁止再输出该段的 `update_tiers` 调整，只能修改 `done=0` 的剩余段位，同时每个被修改的段必须给出新的 target 与 ratio（不可省略比例）。

- **bear_score**: 空头动能。加分项：EMA 空头排列、背驰确认、跌破中枢、供给区派发、上方流动性被扫。
